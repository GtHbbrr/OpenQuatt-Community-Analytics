# Bijdragen aan OpenQuatt Community Analytics 🚀

Wat leuk dat je wilt helpen! Dit project is door en voor de community gebouwd. Of je nu een developer bent, een data-analist of een ervaren installateur: jouw hulp is welkom.

## 🛠 Waar we hulp bij nodig hebben

We zoeken momenteel versterking op de volgende gebieden:

### 1. Data & Analytics (Python / SQL)
- Verfijnen van de **Carnot-efficiëntie** berekening voor verschillende koudemiddelen (R32 focus).
- Ontwikkelen van een model voor **dynamische graaddagen** op de server.
- Normaliseren van data afkomstig van verschillende bronnen (OpenQuatt Modbus vs. HA-integratie).

### 2. Dashboarding (Grafana)
- Bouwen van een 'Community Benchmark' dashboard.
- Visualisaties maken die pendelgedrag en defrost-cycli inzichtelijk maken.
- Ontwerpen van een individueel rapportage-view op basis van de anonieme Device ID.

### 3. ESPHome & Home Assistant (YAML)
- Optimaliseren van de YAML-snippets voor minimale belasting van de ESP32.
- Maken van een Home Assistant 'Blueprint' om deelname voor niet-technische gebruikers makkelijker te maken.

### 4. Infrastructuur
- Meedenken over een robuuste, centrale MQTT-broker en InfluxDB-omgeving.
- Beveiliging van de API-endpoints.

## 🤝 Hoe kun je bijdragen?

1. **Open een Issue:** Zie je een bug of heb je een suggestie voor een nieuwe metric? Laat het weten via de Issues-tab.
2. **Submit een Pull Request:** 
    - Fork de repository.
    - Maak een nieuwe branch (`feature/naam-van-feature`).
    - Commit je wijzigingen.
    - Open een Pull Request met een duidelijke uitleg.
3. **Deel je data:** De simpelste manier om te helpen is door mee te doen! Test de YAML-snippets en geef feedback op de nauwkeurigheid van de berekeningen.

## 📜 Gedragscode & Privacy
We gaan respectvol met elkaar om. Onthoud dat privacy onze hoogste prioriteit heeft: **deel nooit configuratiebestanden waarin persoonlijke gegevens, wachtwoorden of exacte locaties staan.**

Laten we samen de Quatt uit de 'black box' halen!
